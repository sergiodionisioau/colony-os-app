Cross-tenant access attempts fail deterministically
Failures are logged
No data leak occurs

If these paths do not work, Phase 3 is incomplete.

20. Failure conditions

any privileged action can happen without attributable identity
any actor can act outside its scope silently
any secret can leak to logs, audit, or UI unexpectedly

any agent can self-escalate
any module can access undeclared resources
any tenant can observe another tenant’s state
audit logs are incomplete for sensitive operations
revocation does not invalidate future access
