19. 3 critical edge cases
Edge case 1 — Partial environment compatibility during promotion

A release candidate passes staging, but one downstream production-adjacent service or dependency is on a lagging compatible-minor version only.

Required behavior:

detect compatibility gap before promotion
block unsafe promote or require explicit exception gate
show exact impacted dependency
preserve candidate state and evidence
do not allow silent partial rollout
Edge case 2 — Edge rollback with local config preservation

An edge customer upgrades successfully at package verification stage, but application health fails after services come up.

Required behavior:

detect failed post-upgrade verification
trigger rollback
preserve local environment-specific config and customer data expected to survive rollback
restore prior known-good runtime version
generate support-grade evidence bundle
Edge case 3 — Restore technically succeeds but service quality fails

A DR restore completes at the infrastructure and data level, but one required service returns degraded health and violates readiness thresholds.

Required behavior:

do not mark restore as fully successful
block production acceptance/signoff
surface degraded dependency explicitly
require remediation or controlled downgrade of readiness state
keep evidence chain intact

25. Acceptance test for Phase 10

Ask these exact questions.

Deployment test

Can every supported delivery mode be installed, upgraded, rolled back, and supported using a deterministic documented path?

Release test

Can a release be promoted only when evidence-backed gates pass, and rolled back cleanly if it fails?

Continuity test

Can the platform prove recovery through restore and DR drill evidence rather than claims?

Ops security test

Can operational secrets, certs, and privileged deployment actions be rotated, governed, and audited safely?

Support test

Can support and ops teams run the system without engineering improvisation?

Closure test

Can the project be formally closed only when objective technical, operational, and business gates are met?

If any answer is no, Phase 10 is not done.
