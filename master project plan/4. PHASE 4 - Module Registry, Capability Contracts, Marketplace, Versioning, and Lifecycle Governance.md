23. Acceptance test for Phase 4

Ask these exact questions.

Registry test

Can the platform describe a module, its releases, ownership, visibility, and trust status without ambiguity?

Install test

Can a module be installed only when entitlement, trust, compatibility, dependency, permission, and secret requirements all pass?

Upgrade test

Can a module upgrade run with explicit migration review and recover through rollback if verification fails?

Detach test

Can the platform block unsafe detach actions and explain exactly why?

Security test

Can a module be prevented from using undeclared permissions, secrets, connectors, or UI surfaces?

Marketplace test

Can a user understand what a module does, what it needs, what it affects, and whether it is safe to install?

Agent test

Can an agent request module lifecycle changes without bypassing governance, trust, or audit?

If any answer is no, Phase 4 is not done.
