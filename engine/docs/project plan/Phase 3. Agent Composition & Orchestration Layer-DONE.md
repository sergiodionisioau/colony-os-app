## 📦 PHASE 3 OBJECTIVE: Declarative Agent Orchestration

Implement a layer where AI is a pluggable, scoped, and metered component. The kernel remains sovereign; the agent is merely a declarative executor.

---

## 🧾 AGENT TASK SPECIFICATION

Agents interact via the `agent.task_submitted` event:

```json
{
  "type": "agent.task_submitted",
  "payload": {
    "task_id": "uuid",
    "agent_id": "uuid",
    "instruction": "string",
    "constraints": {
      "max_reasoning_steps": 5,
      "max_tokens": 2000,
      "deterministic_mode": false
    }
  }
}
```

## ⚙️ COMPONENT SPECIFICATIONS (RESOLVED)

### 1. Finite Orchestrator
- **Reasoning Loop:** Must use a fixed `for` loop with a hard upper bound (`max_reasoning_steps`). Infinite loops are structurally impossible.
- **Execution Flow:** Agent Receive → AI Reason → Policy Check → Event Emit → Response Await → Loop or End.

### 2. AI Provider Interface (Stateless)
- **Constraint:** Providers MUST NOT store internal state between calls.
- **Protocol:** History is passed explicitly by the Orchestrator in every reasoning call.
- **Plugins:** Supports `null_provider` (stub) and `mock_provider` (testable) in Phase 3.

### 3. Policy Scope Enforcer
- **Mapping:** Agents are bound to a "Policy Scope" (e.g., `business_operator`, `auditor`).
- **Validation:** Every capability call generated by the AI is validated against this scope *before* reaching the Event Bus.

### 4. Agent Crash Isolation
- **Boundary:** Every task execution is wrapped in a `KernelError` try/except boundary.
- **Outcome:** An agent crash or AI failure results in an `agent.task_failed` event, not a kernel panic.

### 5. Memory Adapter Interface
- **Role:** Provides a standard interface for short-term and long-term state.
- **Phase 3:** Includes `null_memory`. No vector DBs or external persistence in this phase.

## 🧱 INTEGRATION REQUIREMENTS

### Event Bus (Phase 2)
- **Correlation:** All events in a task chain MUST share the same `correlation_id`.
- **Replay:** Agent logic must be compatible with replayed events (idempotency).

### Metering (Phase 1)
- **Tracking:** Detailed tracking of AI tokens (simulated) and reasoning steps added to the identity's ledger.

---

## 🛑 ZERO TOLERANCE BASELINE
1. **No Direct Module Calls:** Agents ONLY communicate via the Event Bus.
2. **No Policy Bypass:** AI-generated actions are subject to the SAME RBAC as users.
3. **No Unbounded Loops:** Hard structural limits on reasoning depth.
